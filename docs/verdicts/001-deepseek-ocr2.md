## Что DeepSeek OCR2 заменяет

| Компонент            | Статус     | Причина                              |
|----------------------|------------|--------------------------------------|
| Tesseract OCR        | ✅ ЗАМЕНЁН | 2.5s → 0.7-3s, структура вместо текста |
| Cloud vision (текст) | ✅ ЗАМЕНЁН | 50s → 1-3s, локально, без API         |
| L2 matrix extractor  | ✅ ЗАМЕНЁН | Таблицы → HTML автоматически          |
| L1 block classifier  | ✅ ЗАМЕНЁН | block types: title, text, table, image |

## Что НЕ заменяет

| Компонент         | Статус       | Почему                                       |
|-------------------|-------------|----------------------------------------------|
| VL для image      | ❌ НЕ ЗАМЕНЁН | Bar chart, карта, иерархия — данные визуальны |
| L1 семиотический  | ❌ НЕ ЗАМЕНЁН | topology/spatial/hierarchy — только VL       |
| L3 онтология      | ❌ НЕ ЗАМЕНЁН | Смысловой анализ                             |
| L4 рефлексия      | ❌ НЕ ЗАМЕНЁН | C-level вывод                                |

## Новая архитектура

```
PDF → OCR2 (1-3s/стр, локально, MLX)
        │
        ├── text → L2 Text Extractor (73 стр.)
        ├── table → HTML parser (15 стр.)
        └── image → L1 Classifier Agent → VL Extractor (6-8 стр.)
                                            │
                                            └── qwen3-vl:30b (локально)
```

## Решение

**ПРИНЯТЬ.** DeepSeek OCR2 внедрить как основной нормализатор.

### Выгода

| Метрика         | Было       | Стало       |
|-----------------|------------|-------------|
| Скорость (81 стр.) | ~12 мин    | ~3 мин      |
| Cloud-зависимость | L1+L2 (81 стр.) | Только VL (6-8 стр.) |
| Суверенность    | ⚠️ Cloud   | ✅ Локально |
| Таблицы         | VL-извлечение | HTML автоматически |
| Блоки           | —           | title, text, table, image, figure_title |

### Риски

- OCR2 на сложных image-страницах даёт повторяющийся текст (hallucination) — стр. 6, 37
- Необходим L1 Classifier Agent для различения «image → VL» vs «image → пропустить»
- Модель DeepSeek-OCR-2-bf16-MLX требует ~4 GB RAM
