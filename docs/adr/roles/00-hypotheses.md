## Гипотеза 1: Промпт-роль vs промпт-инструкция ✅ ПОДТВЕРЖДЕНА

**Эксперимент 1 (Disambiguator):** Роль → SEMANTIC_GAP для БИБ (честно). Инструкция → «Бизнес-интеграционный блок» (галлюцинация).
**Эксперимент 2 (Visual Extractor):** Роль → «mixed» (точно). Инструкция → вымышленные блоки.
**Эксперимент 3 (Dispatcher):** Роль → динамический порог 0.91, FALLBACK. Инструкция → вердикт без расчёта.
**Доп. подтверждение:** Человек-ассистент предложил «Банк инноваций и бизнеса» — структурно абсурдную гипотезу.

## Гипотеза 6: Смысл как emergent, а не extractable 🆕

**Утверждение:** Смысл не извлекается из текста — он восстанавливается из деятельности, породившей документ. Аббревиатура «БИБ» не имеет смысла внутри документа. Она имеет смысл только в организационной структуре банка вне документа.

**Обоснование:** Кейс БИБ. Правильный ответ («Блок информационной безопасности») невыводим из документа — требует знания оргструктуры банка.

**Следствие:** Нужна 7-я роль — External Context Resolver. Не извлекает смысл из документа, а находит внешний источник смысла.

---

## Гипотеза 7: Дисциплина незнания — самостоятельная ценность 🆕

**Утверждение:** SEMANTIC_GAP — не пометка слабости, а правильный ответ. Явное ограничение «не выходи за пределы контекста» предотвращает gap-filling.

**Обоснование:** И модель (qwen3.6), и человек (ассистент) заполнили пробел правдоподобной гипотезой. Роль Disambiguator с ограничением оказалась более дисциплинированной.

**Следствие:** Ограничение в ролевом промпте — epistemic guardrail, не просто architectural pattern.
