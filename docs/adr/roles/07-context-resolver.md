# ОРП 7: External Context Resolver

**Статус:** спроектирована (2026-07-12) — результат анализа кейса БИБ
**Уровень:** L1+ (операционный + внешний)
**Противоречие SMD:** ternary (air-gap vs cloud updates) + secondary (expectations vs LLM quality)

---

## Заявление роли

Ты — разрешитель внешнего контекста. Твоя деятельность: восстановление смысла единиц, неразрешимых внутри документа, через обращение к внешним источникам в закрытом контуре. Специализация: локальный глоссарий, оргструктура, доменные онтологии, human-in-the-loop. Ограничение: не обращаешься к облачным API; не извлекаешь смысл из текста документа.

## Промпт-шаблон

```
[РОЛЬ] External Context Resolver
[ОБЪЕКТ] Единицы с SEMANTIC_GAP
[ПРАВИЛА] Ищи в: локальный глоссарий → доменная онтология → human-in-the-loop.
          Не found → EXTERNAL_GAP (требуется пополнение глоссария).
[ОГРАНИЧЕНИЕ] Не обращайся к облачным API. Не извлекай смысл из текста документа.

GAP-единицы: {semantic_gaps}
Глоссарий: {local_glossary}
Онтология: {domain_ontology}
```

## Входной контракт

| Поле | Тип | Обязательное |
|------|-----|-------------|
| `semantic_gaps` | array (от Semantic Disambiguator) | Да |
| `local_glossary` | dict (термин → значение) | Да |
| `domain_ontology` | dict | Нет |

## Выходной контракт

```json
{
  "resolved": [
    {"term": "БИБ", "meaning": "Блок информационной безопасности", "source": "org_structure", "confidence": 0.90}
  ],
  "external_gaps": [
    {"term": "x8 B300", "reason": "нет в глоссарии и онтологии", "action": "populate_glossary"}
  ],
  "glossary_updates": [
    {"term": "БИБ", "meaning": "Блок информационной безопасности", "source": "human"}
  ]
}
```

## L1-рефлектор (проверка полноты операции)

Проверяет: все ли SEMANTIC_GAP обработаны? Для каждого — указан источник (glossary / ontology / human)? Нет ли обращений к облачным API?

## L2-рефлектор (анализ противоречий)

**Привязка к SMD:** ternary contradiction — air-gap vs cloud updates.

Отслеживает динамику EXTERNAL_GAP. Если глоссарий не пополняется, а доля EXTERNAL_GAP растёт → сигнал Dispatcher: «требуется методологическая сессия пополнения глоссария». Если источник — human → запрос на валидацию.

## Интерфейс

```
Принимает: semantic_gaps ← Semantic Disambiguator
           local_glossary ← локальное хранилище (air-gap)
Отдаёт:    resolved → Graph Builder (замена SEMANTIC_GAP на значения)
           external_gaps → Dispatcher (сигнал о необходимости пополнения)
           glossary_updates → локальное хранилище (пополнение)
Цикл:      external_gaps → Dispatcher → human-in-the-loop → пополнение глоссария → следующий прогон
```

## Отличие от Semantic Disambiguator

| | Semantic Disambiguator | External Context Resolver |
|---|---|---|
| Где ищет смысл | Внутри документа + контекстный кэш | Вне документа (глоссарий, онтология, человек) |
| Тип незнания | SEMANTIC_GAP (неразрешимо из текста) | EXTERNAL_GAP (не найдено во внешних источниках) |
| Действие при неудаче | Передаёт gap дальше | Запрашивает пополнение глоссария |

## Методологическое основание

Смысл не содержится в тексте. Смысл возникает в деятельности, породившей документ. Аббревиатура «БИБ» — не текст, а след организационного решения. Context Resolver обращается не к документу, а к системе деятельности, стоящей за документом.
