# ADR-001: Оркестратор с циклами рефлексии вместо линейного пайплайна

**Дата:** 2026-07-10  
**Статус:** принято

## Контекст

Проектируем систему zero-shot парсинга гетерогенных PDF. Документы неизвестны заранее — система не имеет априорных схем, жанровых классификаторов или доменных entity types.

Линейный пайплайн (L0→L1→L2→Pass 2) имеет фундаментальный недостаток: ошибка на раннем слое не может быть исправлена позже. Если L0 неправильно классифицирует регион как Text вместо Diagram, все последующие слои получают искажённые данные.

## Решение

Оркестратор с циклами рефлексии: Agent → Reflector → Agent → ... до стабилизации confidence или достижения лимита итераций.

- **Agent** (qwen3-vl-plus): vision-модель, анализирует изображение/регион документа
- **Reflector** (qwen3.6-35b-a3b): reasoning-модель, проверяет результат агента, находит пробелы и противоречия, формулирует вопросы для следующего прохода
- **Confidence threshold** (0.85): если reflector оценивает результат выше порога — цикл останавливается
- **Max iterations** (3): защита от бесконечных циклов

## Почему не ensemble

Ensemble (多数決) даёт confidence через согласие моделей, но не улучшает качество — три модели могут одинаково ошибаться. Рефлексия же заставляет агента пересмотреть свою работу, найти конкретные пробелы и исправить их.

## Почему не линейный RAG

RAG (retrieve→generate) не решает проблему мультимодальности. Текст, таблицы, диаграммы и изображения требуют разных стратегий извлечения, которые не сводятся к retrieval.

## Последствия

- Каждый цикл добавляет 30-90s latency
- Confidence не всегда растёт с итерациями — глубина анализа растёт, reflector вскрывает новые слои
- Для текстовых документов может быть достаточно одного прохода
- Для сложных диаграмм может потребоваться 2-3 цикла
