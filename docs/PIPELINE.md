# 5-уровневый конвейер обработки документов

**Дата:** 2026-07-20
**Статус:** действующая архитектура

---

## Обзор

Конвейер трансформирует детерминированные данные (PDF) в стохастические (ZoneStore), готовые к применению LLM для C-level анализа.

**Интеграция pipeline:** В коде реализована интеграция двух подходов:
- **roles/ pipeline** (стадии 1-3) — операционная обработка (7 ОРП)
- **semiotic/ pipeline** (стадия 4) — семиотический анализ (L1→L2→L3→L4)

Оба pipeline работают последовательно в `Pipeline.run()`.

```
PDF (детерминированное)
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ Стадия 1: Нормализация (L0) — roles/                        │
│ normalize() из pdf_normalizer.py                            │
│ Вход: PDF-файл                                              │
│ Выход: Universal Representation (JSON)                      │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ Стадия 2: Доменная принадлежность — roles/                  │
│ detect_domain() из domain_analyzer.py                       │
│ Вход: Universal Representation                              │
│ Выход: Domain Info + KGD Result                             │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ Стадия 3: Анализ + Преобразование — roles/ (7 ОРП)          │
│ metadata_extractor, semantic_disambiguator, context_resolver│
│ style_validator, graph_builder                              │
│ Вход: Universal Representation + Domain Info                │
│ Выход: Graph Structure + Metadata + Resolutions             │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ Стадия 4: Семиотический анализ — semiotic/ (L1→L2→L3→L4)   │
│                                                             │
│ L1: classifier.classify_page() → Знаковая форма             │
│     (qwen3-vl:30b, вход: PNG страницы)                      │
│                                                             │
│ L2: extractors.extract_*() → Схема                          │
│     (qwen3-vl:30b, вход: PNG + классификация L1)            │
│                                                             │
│ L3: ontology.map_to_ontology() → Онтология                  │
│     (qwen3.6:35b, вход: JSON-схема от L2)                   │
│                                                             │
│ L4: reflector.reflect() → Рекомендации                      │
│     (qwen3.6:35b, вход: онтология от L3)                    │
│                                                             │
│ Вход: PDF-страницы (PNG) + Universal Representation         │
│ Выход: Classification + Schema + Ontology + Recommendations │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ Стадия 5: Решение — roles/                                  │
│ dispatcher.run()                                            │
│ Вход: Graph + Semiotic Results + Metrics                    │
│ Выход: Решение (итерировать/завершить)                      │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
ZoneStore (стохастическое)
  Эмбеддинги (4096d) + Графы связей + Метаданные + Онтологии
  Готово к LLM-инференсу через ask_orchestrator.py
```

## Уровни детально

### L0: OCR2 — извлечение текста

**Назначение:** Преобразование PDF-страницы в структурированный Markdown с координатами блоков.

**Модуль:** `DeepSeek-OCR2-bf16-MLX` (локальный, через MCP)

**Вход:** PNG-изображение страницы (рендер через PyMuPDF, 200 DPI)

**Выход:** Markdown с тегами:
```
<|ref|>title<|/ref|><|det|>[[x1, y1, x2, y2]]<|/det|>
# Заголовок

<|ref|>text<|/ref|><|det|>[[x1, y1, x2, y2]]<|/det|>
Текстовый блок

<|ref|>table<|/ref|><|det|>[[x1, y1, x2, y2]]<|/det|>
<table>...</table>

<|ref|>image<|/ref|><|det|>[[x1, y1, x2, y2]]<|/det|>
```

**Производительность:** 1-3 секунды на страницу (локальный MLX на Apple M5 Max)

**Кэш:** `/tmp/iafr_ocr_results.json` (81 результат для ИАфр РАН)

---

### L1: SMD-классификатор — 7 знаковых форм

**Назначение:** Классификация страницы по семиотической типологии (СМД Г.П. Щедровицкого).

**Модуль:** `src/semiotic/classifier.py`

**Вход:** Markdown от L0

**Выход:** Одна из 7 знаковых форм:

| Форма | Описание | Пример |
|-------|----------|--------|
| **discursive** | Линейный текст, повествование | Обычный текст, статьи |
| **topology** | Граф, сеть, связи | Организационная структура |
| **matrix** | Таблица, матрица | Финансовые данные |
| **hierarchy** | Иерархия, дерево | Классификация, таксономия |
| **spatial** | Пространственное расположение | Карта, схема |
| **enumeration** | Перечисление, список | Требования, пункты |
| **dynamics** | Процесс, изменение, поток | Workflow, timeline |

**Модель:** qwen3.6:35b (локальная, через Ollama)

**Промпт:** `[РОЛЬ] SMD-классификатор [ОГРАНИЧЕНИЕ] Только 7 форм, без интерпретации`

---

### L2: Экстракторы схем

**Назначение:** Извлечение структурированных схем в зависимости от формы L1.

**Модуль:** `src/semiotic/extractors.py`

**Вход:** Классификация L1 + Markdown L0

**Выход:** Структурированная схема (JSON):

| Форма L1 | Экстрактор | Выход |
|----------|-----------|-------|
| topology | `extract_topology()` | Узлы + рёбра графа |
| matrix | `extract_matrix()` | Строки + столбцы + значения |
| hierarchy | `extract_hierarchy()` | Дерево (parent → children) |
| spatial | `extract_spatial()` | Объекты + координаты + отношения |
| enumeration | `extract_enumeration()` | Нумерованный список |
| dynamics | `extract_dynamics()` | Этапы + переходы + условия |
| discursive | `extract_discursive()` | Параграфы + ключевые тезисы |

**Модель:** qwen3.6:35b (локальная)

---

### L3: Онтологический маппер

**Назначение:** Построение онтологии домена из схем L2.

**Модуль:** `src/semiotic/ontology.py`

**Вход:** Схемы L2 (JSON)

**Выход:** Онтология (JSON):
```json
{
  "entities": [
    {"name": "Сущность", "type": "Тип", "attributes": {...}}
  ],
  "relations": [
    {"from": "Сущность1", "to": "Сущность2", "type": "связь"}
  ],
  "model": "Краткое описание доменной модели"
}
```

**Модель:** qwen3.6:35b (локальная)

**Промпт:** `[РОЛЬ] Онтолог [ОГРАНИЧЕНИЕ] Только сущности и связи из документа, без выдумывания`

---

### L4: Прагматический рефлектор

**Назначение:** Генерация C-level рекомендаций из онтологии L3.

**Модуль:** `src/semiotic/reflector.py`

**Вход:** Онтология L3

**Выход:** Рекомендации (JSON):
```json
{
  "risks": ["Риск 1", "Риск 2"],
  "opportunities": ["Возможность 1", "Возможность 2"],
  "recommended_actions": ["Действие 1", "Действие 2"],
  "confidence": 0.85,
  "urgency": "HIGH"
}
```

**Модель:** qwen3.6:35b (локальная)

**Промпт:** `[РОЛЬ] Прагматический рефлектор [ОГРАНИЧЕНИЕ] Только из онтологии, без внешних знаний`

## Результат конвейера

После прохождения L0-L4 данные сохраняются в **ZoneStore**:

```
ZoneStore (стохастическое представление)
├── Эмбеддинги (4096d, qwen3-embedding:8b)
├── Графы связей (CrossPageLinker)
├── Метаданные (страница, форма, confidence)
└── Provenance (источник, координаты)
```

**Готово к:** LLM-инференсу через `scripts/ask_orchestrator.py`

## Ключевые принципы

1. **Детерминированное → Стохастическое:** Конвейер трансформирует жёсткую структуру PDF в вероятностное представление (эмбеддинги, графы), готовое к LLM.

2. **Локальность:** Все уровни работают на локальных моделях (qwen3.6:35b, qwen3-embedding:8b, DeepSeek-OCR2-MLX). Air-gap capable.

3. **Прослеживаемость:** Каждый результат сохраняет provenance (источник, координаты, confidence).

4. **Инкрементальность:** Каждый уровень может быть выполнен независимо. Можно перезапустить L4 без повторения L0-L3.

5. **Смысловые ходы:** На каждом уровне возникает новый смысл, которого не было на предыдущем уровне. Это не просто обработка данных — это деятельность, порождающая смысл. **Детали:** `docs/SEMANTIC_FLOW.md`
