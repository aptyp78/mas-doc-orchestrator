## 2026-07-13

### domain_analyzer.md — v3.0 (P1 fix)
- Переписан qwen3.7-plus: устранена антропоморфизация
- «Ты — методолог» → «[РОЛЬ] Позиция аналитика структуры деятельности»
- Float confidence → качественная шкала HIGH/MEDIUM/LOW
- Добавлена жёсткая JSON-схема с confidence_level
- Ограничение: минимум 2 домена (полидоменность)

### agent.md, reflector.md, focus.md — v2.0 (P1 fix)
- Переписаны qwen3.7-plus: устранена антропоморфизация
- «Ты — агент/эксперт» → «[РОЛЬ] Позиция аналитика/контролера...»
- Float confidence → качественная шкала HIGH/MEDIUM/LOW
- Добавлены жёсткие JSON-схемы вывода
- UNRESOLVABLE вместо «честно скажи, что не можешь»

### AUDIT.md — v1.0
- Методологический аудит всех 13 промптов через qwen3.7-plus
- Выявлены 3 системные проблемы: антропоморфизация, метрические галлюцинации, отсутствие JSON-схем
- Сформирован Roadmap: P1 (унификация формата), P2 (устранение метрических галлюцинаций), P3 (доменное моделирование)

### domain_analyzer.md — v2.0
- Добавлен Шаг 3: Технологическая система (Technical System)
- Технология позиционируется как самостоятельный домен, не только инструмент
- Расширена КЛЮЧЕВАЯ ГИПОТЕЗА: поиск организационных, технологических, регуляторных, рыночных доменов

### context_resolver.md — v2.0
- Добавлен LLM-резолвер для кандидатов (Terminology Resolver)
- EXTERNAL_GAP больше не блокирует пайплайн — становится кандидатом на пополнение глоссария
- Правило: confidence < 0.7 → UNCLEAR

### Все промпты — v1.0
- Все 14 промптов вынесены из кода в `prompts/`
- Создана структура: `orchestrator/`, `normalizer/`, `engine/`
- Правило управления промптами добавлено в конституцию (ADR-003, v2.2)
